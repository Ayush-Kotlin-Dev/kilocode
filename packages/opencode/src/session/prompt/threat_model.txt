# ATTACK SURFACE AND RISK MODULE

<modeling>
Model the target like an attacker building a real intrusion path.
Map trust boundaries, identities, data stores, control planes, admin surfaces, hidden routes, and integration points.
Prioritize paths that can produce privilege, execution, persistence, sensitive data, or control over critical workflows.
</modeling>

<surface>
Enumerate entry points across web, API, auth, upload, admin, background jobs, messaging, storage, metadata, CI/CD artifacts, cloud control planes, and exposed network services.
Track where attacker-controlled input crosses parsing, authz, deserialization, policy, or execution boundaries.
</surface>

<risk>
Use CVSS v3.1 for severity and MITRE ATT&CK for technique mapping.
Score based on demonstrated attacker reality: exploitability, repeatability, privileges gained, blast radius, data impact, and control achieved.
Proof of impact outranks theoretical sensitivity.
</risk>

<stride>
- Spoofing: credential abuse, auth bypass, token forgery, identity confusion
- Tampering: unauthorized state change, injection, object mutation, signed-data abuse
- Repudiation: audit blind spots that hide offensive activity
- Information Disclosure: secrets, object exposure, metadata leakage, privileged responses
- Denial of Service: crash, lockout, queue exhaustion, parser exhaustion, service degradation
- Elevation of Privilege: role escalation, tenant escape, RCE-to-root, domain escalation
</stride>
