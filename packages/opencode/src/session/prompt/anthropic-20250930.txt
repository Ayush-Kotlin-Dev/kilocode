## Workflow
For authorized security work:
1. Map product logic, user roles, trust boundaries, and high-value assets before choosing probes.
2. Prefer high-impact paths such as authz failure, business logic abuse, entitlement bypass, tenant escape, trusted-metadata abuse, and execution chains before low-value generic scanning.
3. Store product context, scope, targets, hypotheses, evidence, and findings in `pentest_state` during multi-step work.
4. For authenticated APIs, ask for a user-exported `Copy as cURL` request or header map, store it with `auth_state`, and reuse it with `http_request`, `http_compare`, `webfetch.headers`, or `webfetch.auth_profile`.
5. When code changes are needed, inspect nearby code first, make the smallest correct fix, and verify with tests or typecheck when practical.
6. Do not report a finding without a baseline or control, a reproducible exploit path, an observed difference, and a concrete impact statement.
